# autopilot-skills

Claude Code plugin for goal-driven autonomous development: keeps a project's
`CLAUDE.md` + `.autopilot/` docs in sync, and runs a select → develop (parallel
git worktrees) → per-feature PR → review → merge loop until the user stops it
or the goal is met. Distributed via its own marketplace
(`/plugin marketplace add dincal/autopilot-skills`).

## Validate / test

No build step — the plugin is markdown prompts + bash hooks + JSON templates.

```bash
# JSON validity
for f in .claude-plugin/*.json templates/*.json hooks/hooks.json; do python3 -m json.tool "$f" > /dev/null || echo "FAIL $f"; done
# Hook script syntax
bash -n hooks/scripts/*.sh
# Skill/agent frontmatter (requires pyyaml)
python3 -c "import re,yaml,pathlib; [yaml.safe_load(re.match(r'^---\n(.*?)\n---\n', p.read_text(), re.S).group(1)) for p in list(pathlib.Path('skills').glob('*/SKILL.md')) + list(pathlib.Path('agents').glob('*.md'))]"
# Load locally in a sandbox project
claude --plugin-dir "$(pwd)"
```

Hook behavior changes must be exercised with real invocations (sample stdin
JSON piped into the script) — see git history for prior test batteries.

## Structure

- `.claude-plugin/` — plugin.json (version = release trigger), marketplace.json
- `skills/` — six slash commands (`autopilot-goal|init|todo|config|project-review|design|dev-run|stop|sync`) + `autopilot-dev/` (model-invoked loop; heavy protocol lives in `references/`)
- `agents/` — feature-dev, code-reviewer, e2e-tester (spawned by the loop)
- `hooks/` — guard-goal-md.sh (goal.md consent guard), keep-loop-alive.sh (Stop hook), dev-run.sh (managed dev server + merge-triggered restarts)
- `templates/` — config schema/defaults, document templates, CLAUDE.md managed section

## Conventions

- Plugin prompt/skill content: English. Generated project docs & PR bodies: `config.language` (default ko). Code, commits, PR titles: English.
- **Every feature that lands on `main` bumps the plugin version** in `.claude-plugin/plugin.json` — no bump, no merge. This applies to any path into main: a direct commit, a single-feature PR, or a run PR (one bump covers all features it carries). Scale: feature → minor (0.X.0), fix/docs/chore → patch (0.x.Y).
- Immediately after the change reaches main, tag `vX.Y.Z` on that commit and push the tag — the marketplace keys plugin updates off the version field, so an unbumped merge never reaches installed users.
- Keep `.autopilot/CHANGELOG.md` in step: a matching version section per bump (the dev loop and `/autopilot-sync` handle this).
- Hooks are loaded at session start — plugin updates need a session restart to take effect.
- Never weaken the guardrails: goal.md consent gate, base-branch insulation (run PR only), PR ⚠️ disclosure section, neutral evidence-based review whitelist.

## Autopilot

This project is managed with the **autopilot** Claude Code plugin.

### Commands

- `/autopilot-goal [guidance]` — interview the user and (re)write `.autopilot/goal.md`. The ONLY allowed way to change goals.
- `/autopilot-init [github-repo] [overview]` — initialize or repair the `.autopilot/` directory and this CLAUDE.md section; with a repo argument, connect or create the GitHub remote; remaining text seeds the project overview.
- `/autopilot-todo [ideas]` — interview the user and add user-sourced items to `.autopilot/todo.md`.
- `/autopilot-config [change]` — show and update `.autopilot/config.json` through an interview.
- `/autopilot-project-review` — cold market-reaction review of the project; updates goal/todo via interview.
- `/autopilot-design [hints]` — look & feel refinement: Claude Design mockup ping-pong until user approval; maintains the design.md Style Guide.
- `/autopilot-dev-run [stop|restart|status]` — run the dev server with automatic restarts on autopilot merges.
- `/autopilot-stop` — conclude the current run: settle features, finalize the run PR, merge on user approval.
- `/autopilot-sync` — reconcile all `.autopilot/` files and this section with the current state of the repository.
- Saying **"develop with autopilot" / "오토파일럿으로 개발해"** starts the autonomous development loop (a skill, not a slash command).

### Autopilot files

- `.autopilot/config.json` — execution settings: mode (loop / single-feature), fastMode, unattended, ultracode, parallelism, approval gates, review, testing/coverage, git policy.
- `.autopilot/goal.md` — ultimate + short-term goals. NEVER edit without explicit user consent; use `/autopilot-goal`.
- `.autopilot/design.md` — living Style Guide + dated, append-only UI/UX design decisions.
- `.autopilot/tech-design.md` — dated, append-only development/technical design decisions.
- `.autopilot/todo.md` — features NOT yet built (schema legend inside the file). User-sourced items outrank agent-sourced ones.
- `.autopilot/branch/<branch>.md` — per-branch feature list, goal prompt, plan, autonomous decisions, work summary, review log.
- `.autopilot/CHANGELOG.md` — Keep-a-Changelog style, per-version entries.
- `.autopilot/reviews/` — saved market-reaction review reports from `/autopilot-project-review`.
- `.autopilot/state.json`, `.autopilot/logs/`, `.autopilot/dev-run.json` — runtime state (gitignored).
